# Talkute 产品体验设计方案

> 基于 TalkuteUX.md 的 UX/UI 描述，结合现有 Flutter + Rust 技术架构，将设计理念落地为可执行的产品体验规格。

---

## 一、设计总纲

### 设计哲学

Talkute 的界面遵循三个字：**克制、轻量、系统原生**。核心价值在 AI 算力，界面绝不喧宾夺主。用户感知到的不是一个"App"，而是一个随时待命的隐形助手——按下快捷键，说话，松手，精炼文字自动上屏。整个过程的认知负荷趋近于零。

### 品牌视觉语言

| 属性 | 规格 |
|------|------|
| 主色 (Primary) | `#4F46E5` (Indigo-600)，传达专业、智能、信赖 |
| 录音态强调色 | `#EF4444` → `#F97316` 渐变，传达"正在倾听"的温暖感 |
| 成功色 | `#10B981` (Emerald-500)，用于"上屏成功"的瞬间反馈 |
| 背景色 (Light) | `#FAFAFA` 主背景 / `#FFFFFF` 卡片 |
| 背景色 (Dark) | `#0F0F0F` 主背景 / `#1A1A1A` 卡片 |
| 毛玻璃效果 | 桌面端胶囊窗使用 `BackdropFilter`，sigma: 20，opacity: 0.85 |
| 字体 | 系统原生字体栈：Windows 用 Segoe UI，macOS 用 SF Pro，Linux 用 Inter |
| 圆角 | 胶囊窗 24px，卡片 16px，按钮 12px，输入框 8px |
| 动效曲线 | `Curves.easeOutCubic`，时长 200-400ms，拒绝生硬的线性动画 |

## 二、桌面端体验设计 (Windows / macOS / Linux)

桌面端是 Talkute 的主战场。设计模式为"隐形助手"——平时完全不可见，快捷键唤起后以极低存在感完成工作。

### 2.1 系统托盘 (System Tray)

常驻于系统通知区域（Windows 右下角 / macOS 右上角状态栏），是 Talkute 的"锚点"。

**托盘图标状态：**

| 状态 | 图标表现 | 说明 |
|------|---------|------|
| 待机 | 品牌 Logo（单色，与系统图标融合） | 静默存在，不打扰 |
| 录音中 | Logo 叠加红色脉冲圆点 | 让用户在任何时候都能确认"它在听" |
| 处理中 | Logo 叠加旋转弧线 | AI 正在润色 |
| 错误 | Logo 叠加黄色三角 | 需要注意 |

**托盘右键菜单（卡片式 UI）：**

```
┌─────────────────────────────────┐
│  Talkute                   v1.0 │
│─────────────────────────────────│
│  ● 就绪                         │
│                                 │
│  ┌─ 今日用量 ─────────────────┐ │
│  │  ████████░░  1,247 / 5,000  │ │
│  │  字（免费额度）              │ │
│  └─────────────────────────────┘ │
│                                 │
│  ⌨  快捷键: Ctrl+Shift+Space   │
│  🕐 闲置上屏: 1.5 秒  [━━●━━]  │
│  🌐 语言: 自动检测              │
│                                 │
│─────────────────────────────────│
│  ⚙ 偏好设置                     │
│  📋 历史记录                     │
│  ❓ 帮助                         │
│─────────────────────────────────│
│  退出 Talkute                    │
└─────────────────────────────────┘
```

**关键交互细节：**

闲置时间滑块是核心设置，直接暴露在托盘菜单中而非深埋在设置页。用户拖动滑块（范围 0.5s - 5.0s，步进 0.5s），实时预览数值变化。这个值决定了"用户停止说话后多久自动上屏"，是影响体验流畅度的最关键参数。

### 2.2 悬浮胶囊窗 (Floating Capsule)

这是 Talkute 桌面端的核心交互界面——一个极其克制的悬浮窗。

**唤起方式：** 用户按下全局快捷键（默认 `Ctrl+Shift+Space` / macOS `⌘+Shift+Space`）。

**出现位置：** 屏幕正中偏下（距底部 1/4 处），不跟随鼠标，避免遮挡正在编辑的文本区域。

**胶囊窗布局（录音态）：**

```
    ┌──────────────────────────────────────────┐
    │                                          │
    │   🔴  ▁▃▅▇▅▃▁▃▅▇▅▃▁   中文 → 中文      │
    │                                          │
    └──────────────────────────────────────────┘
         ↑        ↑                  ↑
      状态灯   动态声波          语言指示
```

**尺寸规格：** 宽 360px，高 64px，圆角 32px（完美胶囊形），毛玻璃背景。

**胶囊窗的四个状态：**

**① 录音态 (Recording)**
- 左侧：红色呼吸脉冲圆点（直径 12px，opacity 在 0.4-1.0 间循环，周期 1.2s）
- 中间：实时声波动画（16 根竖线，高度随音量实时变化，品牌色渐变）
- 右侧：当前语言标签（如"中文 → 中文"或"中文 → English"）
- 整体：胶囊边框有微弱的品牌色发光（box-shadow，blur 20px，opacity 0.3）

**② 处理态 (Processing)**
- 声波冻结为最后一帧形态
- 红色圆点变为品牌色旋转弧线（加载指示器）
- 出现文字提示："AI 润色中..."，使用 shimmer 效果
- 持续时间通常 < 1 秒，用户几乎无感

**③ 完成态 (Done)**
- 胶囊短暂闪烁成功色（`#10B981`），持续 600ms
- 显示"✓ 已上屏"文字，然后胶囊以 `fadeOut + scaleDown` 动画消失（300ms）
- 文字已自动注入到用户光标所在位置

**④ 错误态 (Error)**
- 胶囊变为警告色边框
- 显示简短错误信息（如"未检测到麦克风"）
- 3 秒后自动消失，或用户点击关闭

**交互逻辑：**

快捷键的行为模式支持两种，用户可在设置中切换：

模式 A "按住说话"（Push-to-Talk）：按住快捷键开始录音，松开后自动触发 AI 处理并上屏。适合短句输入。

模式 B "切换录音"（Toggle）：按一次开始录音，再按一次结束。结束后等待闲置时间到达后自动上屏。适合长段落口述。默认使用此模式。

### 2.3 结果预览与编辑（可选高级功能）

在设置中可开启"上屏前预览"模式。开启后，AI 处理完成时胶囊窗会向下展开为一个预览卡片：

```
    ┌──────────────────────────────────────────┐
    │   ✓ AI 润色完成                    ✕     │
    ├──────────────────────────────────────────┤
    │                                          │
    │   "我认为这个方案在技术可行性和成本       │
    │    控制方面都具有明显优势，建议团队        │
    │    优先推进第二阶段的实施。"              │
    │                                          │
    ├──────────────────────────────────────────┤
    │   [ 重新录制 ]    [ 编辑 ]    [✓ 上屏 ]  │
    └──────────────────────────────────────────┘
```

预览卡片尺寸：宽 400px，高度自适应内容，最大高度 300px（超出滚动）。三个操作按钮：重新录制（丢弃当前结果重新开始）、编辑（进入文本编辑模式，用户可手动修改）、上屏（确认注入）。默认关闭此功能，直接上屏，保持"无感"体验。

## 三、移动端体验设计 (iOS / Android)

移动端 Talkute 作为系统输入法运行，替代传统键盘。

### 3.1 键盘面板布局

摒弃传统 26 键布局，采用极简设计：

```
┌──────────────────────────────────────────────────┐
│  🌐 中→中  │  🔄 翻译  │  📖 字典  │  ⚙ 设置   │  ← 快捷工具栏
├──────────────────────────────────────────────────┤
│                                                  │
│                                                  │
│              ▁▃▅▇▅▃▁▃▅▇▅▃▁                      │  ← 动态声波区
│              (录音时显示)                          │
│                                                  │
│                 ┌──────┐                         │
│                 │  🎤  │                         │  ← 巨大麦克风按钮
│                 │      │                         │
│                 └──────┘                         │
│                                                  │
│           点击录音 / 长按录音                      │
│                                                  │
├──────────────────────────────────────────────────┤
│  ⌨ 切换键盘  │           │  ⏎ 换行  │  ⌫ 删除   │  ← 底部辅助栏
└──────────────────────────────────────────────────┘
```

**麦克风按钮规格：**

| 属性 | 规格 |
|------|------|
| 尺寸 | 直径 80dp（静止态）→ 96dp（录音态） |
| 颜色 | 品牌色渐变（`#4F46E5` → `#6366F1`） |
| 录音态 | 渐变变为录音色（`#EF4444` → `#F97316`），外圈涟漪扩散动画 |
| 阴影 | 静止态 elevation 4dp，录音态 elevation 12dp + 品牌色发光 |
| 交互 | 支持点击切换（Toggle）和长按说话（Push-to-Talk）两种模式 |
| 触觉反馈 | 开始录音时轻触觉（HapticFeedback.lightImpact），结束时中触觉 |

**动态声波区：**

录音时，麦克风按钮上方显示实时声波动画。声波由 24 根竖线组成，高度随麦克风音量实时变化，颜色使用品牌色到录音色的渐变。声波区高度 60dp，宽度占键盘面板宽度的 70%，居中显示。

**快捷工具栏：**

位于键盘面板顶部，高度 44dp，水平排列 4 个图标按钮。语言切换按钮显示当前语言对（如"中→中"），点击弹出语言选择器。翻译模式开关点击后高亮，表示"说 A 语言，上屏 B 语言"。字典按钮进入自定义词库管理。设置按钮进入偏好设置。

**底部辅助栏：**

保留系统键盘切换按钮（用户可随时切回系统键盘）、换行键和删除键，确保基本文本编辑能力。

### 3.2 录音过程中的全屏反馈

当用户长按麦克风按钮进入录音时，键盘面板可选择性地进入"沉浸录音模式"：

整个键盘区域背景渐变为深色（`#0F0F0F`，opacity 0.95），声波动画放大至全宽显示，底部显示实时转录的文字预览（浅色半透明文字，让用户知道"它听到了什么"）。这种沉浸感让用户更专注于表达，减少"收音焦虑"。

### 3.3 上屏动效

AI 处理完成后，文字上屏的瞬间：麦克风按钮短暂变为成功色（`#10B981`）并显示 ✓ 图标，持续 400ms 后恢复。同时给予一次成功触觉反馈（`HapticFeedback.mediumImpact`）。整个过程用户无需任何额外操作。

## 四、偏好设置界面设计

设置界面采用分组卡片布局，移动端为全屏页面，桌面端为独立窗口（600×800px）。

### 4.1 设置分组结构

**第一组：语音输入**
- 输入模式：点击切换 / 长按说话（单选）
- 闲置上屏时间：滑块（0.5s - 5.0s），实时预览数值
- 麦克风选择：下拉列表（桌面端显示所有可用音频输入设备）
- 降噪：开关（默认开启）

**第二组：AI 润色**
- 润色强度：三档选择卡片——轻度（仅去口头禅和标点）/ 标准（语法纠错 + 逻辑整理）/ 深度（完整重写为书面语）
- 语气风格：专业 / 日常 / 学术 / 自动适配（根据目标应用自动切换）
- 上下文感知：开关（开启后根据当前应用类型自动调整输出风格）

**第三组：语言与翻译**
- 输入语言：自动检测 / 手动指定
- 输出语言：与输入相同 / 指定目标语言
- 翻译模式：开关 + 语言对选择

**第四组：自定义字典**
- 双列列表布局：左列"语音发音/昵称"，右列"标准词汇"
- 示例行：「泰普勒斯 → Talkute」「杰皮提 → GPT」
- 支持添加、编辑、删除、导入/导出

**第五组：快捷键（仅桌面端）**
- 全局唤起：默认 `Ctrl+Shift+Space`，支持自定义
- 快速切换语言：默认 `Ctrl+Shift+L`
- 切换翻译模式：默认 `Ctrl+Shift+T`

**第六组：隐私与数据**
- 本地处理模式：开关（开启后所有数据仅在本地处理）
- 历史记录保留：关闭 / 7天 / 30天 / 永久
- 清除所有数据：确认按钮

**第七组：关于**
- 版本号、开源许可、反馈入口


### 6.3 跨语言翻译流程

当用户开启翻译模式时，流程增加一步：

```
语音输入（中文）
    → ASR 转录（中文文本）
    → NLP 润色（中文书面语）
    → LLM 翻译（英文）
    → 上屏（英文）
```

胶囊窗/键盘面板的语言标签显示为"中文 → English"，让用户始终清楚当前的语言对。

## 七、动效与微交互规格

### 7.1 动效清单

| 动效名称 | 触发时机 | 规格 | 目的 |
|---------|---------|------|------|
| 胶囊淡入 | 按下快捷键 | fadeIn + scaleUp(0.8→1.0)，200ms，easeOutCubic | 轻盈出现，不突兀 |
| 录音脉冲 | 录音中持续 | 红色圆点 opacity 0.4↔1.0，1200ms 循环 | 确认"正在听" |
| 声波律动 | 录音中持续 | 16根竖线高度随音量变化，60fps，弹性阻尼 | 音量可视化反馈 |
| 涟漪扩散 | 移动端录音中 | 从按钮中心向外扩散的同心圆，opacity 递减 | 增强录音感知 |
| 处理旋转 | AI 处理中 | 弧线旋转，800ms/圈 | 表示正在工作 |
| Shimmer | AI 处理中文字 | 光泽从左到右扫过文字，1500ms 循环 | 优雅的加载态 |
| 成功闪烁 | 上屏完成 | 边框/按钮变为成功色，600ms 后恢复 | 确认完成 |
| 胶囊消失 | 上屏完成后 | fadeOut + scaleDown(1.0→0.8)，300ms | 优雅退场 |
| 按钮缩放 | 按下麦克风 | scale 1.0→0.95→1.0，100ms | 按压反馈 |

### 7.2 触觉反馈（移动端）

| 时机 | 强度 | API |
|------|------|-----|
| 开始录音 | 轻 | `HapticFeedback.lightImpact` |
| 结束录音 | 轻 | `HapticFeedback.lightImpact` |
| 上屏成功 | 中 | `HapticFeedback.mediumImpact` |
| 错误 | 重 | `HapticFeedback.heavyImpact` |

### 7.3 音效（可选，默认关闭）

开始录音：轻柔的"叮"声（高频，100ms）。结束录音：低沉的"咚"声（低频，150ms）。上屏成功：清脆的"嗒"声（中频，80ms）。

## 八、平台适配策略

### 8.1 Windows

- 使用 `fluent_ui` 包实现 Fluent Design 风格
- 系统托盘通过 Rust `system_tray` 模块实现（已有骨架）
- 全局快捷键通过 Rust `global_hotkey` 模块实现（已有骨架）
- 悬浮胶囊窗使用 Flutter 的无边框透明窗口（`window_manager` 包）
- 文本注入通过 Rust `text_injection` 模块模拟键盘输入（已有骨架）
- 亚克力/云母材质效果使用 `flutter_acrylic` 包

### 8.2 macOS

- 使用 `macos_ui` 包实现原生 macOS 风格
- 状态栏菜单通过 Rust `system_tray` 模块实现
- 全局快捷键通过 Rust `global_hotkey` 模块实现
- 悬浮窗使用 NSPanel 级别的透明窗口
- 毛玻璃效果使用系统原生 `NSVisualEffectView`

### 8.3 Linux

- 使用 `yaru` 包实现 Ubuntu 风格（可选 Adwaita）
- 系统托盘通过 AppIndicator / StatusNotifierItem 实现
- 全局快捷键通过 X11/Wayland 协议实现
- 悬浮窗使用 GTK Layer Shell（Wayland）或 X11 override-redirect

### 8.4 iOS / Android

- 使用 Material 3 设计语言
- 作为自定义输入法（IME）运行
- iOS 通过 App Extension (Custom Keyboard) 实现
- Android 通过 InputMethodService 实现
- 移动端暂不支持系统托盘和全局快捷键，通过键盘面板内的 UI 完成所有交互

## 九、现有代码与 UX 设计的差距分析

### 9.1 需要重构的部分

| 现有实现 | UX 要求 | 改动 |
|---------|---------|------|
| `HomePage` 是传统全屏 App 布局 | 桌面端应为"隐形"模式，无主窗口 | 桌面端需重构为托盘 + 胶囊窗模式 |
| `RecordingButton` 是简单的圆形按钮 | 需要涟漪扩散、品牌色渐变、状态切换动效 | 需要增强动效和视觉表现 |
| `WaveformWidget` 使用随机数据 | 需要接入真实音频数据，实时响应 | 需要与 Rust audio 模块联通 |
| `SettingsPage` 是基础列表 | 需要分组卡片、滑块、自定义字典等 | 需要大幅扩展 |
| 无悬浮窗实现 | 桌面端核心交互是悬浮胶囊窗 | 需要新增 `FloatingCapsule` 组件 |
| 无系统托盘 UI | 托盘菜单是桌面端锚点 | 需要新增 `TrayMenu` 组件 |
| 无翻译模式 UI | 语言对选择和翻译开关 | 需要新增翻译相关 UI |
| 无自定义字典 UI | 双列列表编辑界面 | 需要新增 `DictionaryPage` |

### 9.2 可复用的部分

现有的 `EngineService`、`ConfigService`、Riverpod 状态管理架构、`AppConfig` 数据模型（已包含 `DictionaryEntry`）、平台适配的 `TalkuteApp` 壳子，以及 Rust 侧的 `system_tray`、`global_hotkey`、`text_injection` 模块骨架，都可以在新设计中直接复用或扩展。
